# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A static, single-page web app for practicing the Cundī Dhāraṇī (準提咒), a Buddhist mantra. It's entirely client-side HTML/CSS/JS — no build step, no package manager, no server code. The whole app lives in one file: `index.html`.

## Development workflow

There is no build/lint/test tooling in this repo (no `package.json`, no bundler, no test runner). To work on this project:

- Edit `index.html` directly — its `<style>` and `<script>` blocks are inline.
- Preview by opening `index.html` in a browser (or serving the directory with any static file server, e.g. `python3 -m http.server`). The Firebase visitor-counter script uses ES module imports over HTTPS CDN URLs, so `file://` may block it in some browsers — prefer serving over `http://localhost`.
- There is no automated test suite. Manually verify changes in a browser: click through all three modes (逐句練習/counting/迴向文), play audio clips, and check the completion-share flow.

## File structure

- `index.html` — the live, current version of the app. Treat this as the source of truth.
- `Index2.html` — an earlier draft/superseded copy of the app (uses `bg.jpeg` instead of `completion-bg.png`, missing favicon links). Not linked from anywhere and not actively maintained — don't assume changes to `index.html` need mirroring here unless asked.
- `test.html` — a trivial static page used to sanity-check that CSS renders (unrelated to the mantra app).
- `icon.png`, `completion-bg.png` — assets actually referenced by `index.html` (favicon and the completion-share card background).
- `bg.jpeg`, `bg.JPG`, `BA477F14-*.png`, `IMG_9093.jpeg`, `IMG_9099.jpeg` — image assets not currently referenced by `index.html` (some are used only by `Index2.html`); treat as legacy/unused unless you trace a reference.
- `新錄音 *.m4a`, `Test.m4a`, `Test 2.mp3` — local audio recordings checked into the repo. **`index.html` does not play these local files** — see below.

## Architecture of `index.html`

Everything is in one file, organized into three parts:

1. **Inline `<style>`** — all CSS for the three UI modes (phrase practice, mantra counter, dedication text) and the completion/share modal.
2. **`<script>` (classic, non-module)** — the app logic:
   - `phrases[]` is the single source of truth for mantra content: each entry has `section`, `han` (Chinese), `zhuyin`/`san` (pronunciation), `note`, and optionally `audio` (a URL). UI for phrase practice, the full-text reference list, and the counting-mode mantra list are all rendered from this one array — don't hand-edit the DOM in multiple places when adding/editing a phrase, edit `phrases[]`.
   - Custom `<audio>` player wiring (`wireCustomPlayer`) drives the practice-mode play button/progress bar; a separate plain `Audio()` instance (`miniAudioPlayer`) drives the small per-line play buttons in counting mode.
   - The mantra counter, goal selection, and completion celebration (`showCompletionScreen`) are plain DOM/state code (`total`, `goal` variables).
   - The completion **share image** is generated by drawing onto a `<canvas>` (`buildCompletionImageBlob`) at 3x scale, then shared via the Web Share API (`navigator.share`) with a download-link fallback for browsers that don't support sharing files.
3. **`<script type="module">`** — Firebase (Firestore) integration for the site-wide visitor/mantra-tap counters shown in the footer. Reads/writes four documents in a `stats` collection (`visitor_total`, `day_<date>`, `mantra_total`, `completed108_total`) and exposes `window.recordMantraTap` / `window.recordCompletion108`, which the counting-mode code calls. The Firebase config object (apiKey, projectId, etc.) is the public client config for a Firebase web app — this is normal for Firebase and not a leaked secret.

## Key conventions / gotchas

- **Audio is hosted cross-repo.** The `audio` URLs in `phrases[]` point to `https://raw.githubusercontent.com/k740922/Cundi--audio/main/...` — a *different* GitHub account/repo than this one (`iamheremama/Cundi--audio`). The `.m4a` files checked into *this* repo are not what's actually played by the live page. If you add/replace a recording, either update the URL to wherever the file truly lives, or be aware you're only changing an unused local copy.
- Filenames in that audio path are Chinese ("新錄音 N.m4a") and must stay percent-encoded in the URL (e.g. `%E6%96%B0...`).
- All user-facing text is Traditional Chinese (zh-Hant). Match the existing tone (formal/devotional) and font stack (`Noto Serif TC` for headings, `Cormorant Garamond` for Sanskrit/pinyin, `Noto Sans TC` for body/UI) when adding UI text.
- The counting/goal/completion state (`total`, `goal`, `goalCelebrated`) is in-memory only — it resets on page reload; only the aggregate Firestore counters persist across sessions.
